# Session Log — 2026-02-23 Print Fix

**Date:** 2026-02-23  
**Focus:** Fix print-pdf detection for SSG

## Summary

The `?print-pdf` query parameter detection was on the server side (`Astro.url.searchParams`), which always evaluates to `false` during static site generation. Dallas moved detection to client-side JavaScript:

- **`slide-viewer.tsx`** — auto-detects `?print-pdf` in `window.location.search` and overrides `embedded` to `false` so reveal.js renders in standalone print layout.
- **`[slug].astro`** — added `initPrintMode()` client-side function that checks for `?print-pdf`, hides header/instructions, and applies full-viewport CSS classes.

Single-agent fix, no design or review spawn needed.
